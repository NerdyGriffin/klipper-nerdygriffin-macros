# Save Config (save_config.cfg)

The `save_config.cfg` provides utilities for safely saving printer configuration with automatic cleanup and conditional scheduling.

## Usage

```gcode
SAVE_CONFIG                         # Save config with automatic cleanup
SAFE_SAVE_CONFIG                    # Save config with safety checks (skips SAVE_CONFIG during print, cools extruder to 60°C before saving)
SET_COMPLETE_SAVE_CONFIG [ENABLE=1] # Enable/disable post-print save
GET_COMPLETE_SAVE_CONFIG            # Check if post-print save is enabled
```

### SET_COMPLETE_SAVE_CONFIG Parameters

| parameters | default value | description |
|-----------:|---------------|-------------|
| ENABLE | 1 | Set to 1 to enable post-print save, 0 to disable |

## Behavior

- `SAVE_CONFIG` (overrides Klipper's built-in command) automatically removes default beacon model and bed mesh profiles before saving to prevent clutter in the autogenerated config section
- `SAFE_SAVE_CONFIG` checks if a job is active, cools extruder to 60°C, then calls `SAVE_CONFIG`
- `SET_COMPLETE_SAVE_CONFIG` schedules automatic save after print completion
- `CONDITIONAL_SAVE_CONFIG` executes the scheduled save (called automatically by `PRINT_END`)

## Automatic Cleanup

Before saving, `SAVE_CONFIG` automatically:
- Removes the "default" beacon model (if it exists in runtime or config)
- Removes the "default" bed mesh profile (if it exists)
- Clears the active bed mesh

This prevents temporary calibration data from cluttering the `#*# <--- SAVE_CONFIG --->` section of your config file.

## Internal Macros

These macros are called automatically and should not be called directly:

- `CONDITIONAL_SAVE_CONFIG` - Executes scheduled save if enabled
- `delayed_save_config` - Delayed G-code for deferred save execution
